SmartWinnr is a revenue-enablement platform: learning, assessment, coaching,
and gamification in one place. Use this overview to figure out which
modules your org has - and then jump to the docs that match.

## What you can do with SmartWinnr

- **Learn fast** - bite-sized SmartFeeds, structured SmartPaths, and
  knowledge-hub content delivered to learners on web and mobile.
- **Assess knowledge** - create quizzes, run question pools, and read
  per-learner and per-team analytics.
- **Coach in 1:1** - review video-coaching submissions, run AI role-plays,
  or capture field-coaching observations against templated competencies.
- **Gamify outcomes** - competitions, scorecards, leaderboards, badges,
  XP, streaks.
- **Collect feedback** - surveys and forms with reviewer workflows.

## How this documentation is laid out

Every module follows the same shape under
[**Modules**](/modules/):

| Sub-folder | Who it's for |
|---|---|
| **For Learners** | What end users do day-to-day |
| **For Managers** | Reviewing reportees, reading team analytics |
| **Create & Manage** | Authoring CRUD - for editors and admins |
| **Assign & Schedule** | Distribution to users, groups, metatags |
| **Features** | Per-feature reference |
| **Reports & Analytics** | Admin-level reports and dashboards |
| **Settings & Permissions** | Module configuration |
| **FAQs & Troubleshooting** | Common questions, error codes |

You only see the sub-folders your role and your org's privileges allow.
A learner browsing the Quiz module sees **For Learners** + **FAQs**; an
editor sees the full shape.

## Which modules does your org have?

Module availability is set per organization. If you open a module page and
see a "Talk to your admin to enable" upsell instead of the get-started
cards, your org hasn't licensed that module yet - the docs are still
readable, but the feature isn't live for you.
