Original IA comparison (kept for reference)

The three high-level options we evaluated before settling on Option B. Click through if you want to revisit the decision.

Option A · Lowest risk

Role-Aware Topic Shelves

Top-level mixes audience (For Learners / For Managers / Authoring & Admin) with cross-cutting topics (Reports, Integrations, System Admin, Reference). Closest evolution of today's structure — every customProps.roles and customProps.privilege gate keeps working unchanged.

Pros
  • Familiar to existing editors and admins.
  • Reuses today's role/privilege gating with no logic changes.
  • Cheapest to ship — sidebar reorg + per-module sub-sectioning.
Cons
  • Still mixes "who" and "what" in the same level — slightly less crisp.
  • Cross-cutting topics (Reports, KPIs) compete with audience shelves for attention.

Option B · Most modern

Pure Topic Taxonomy (ClickHouse-style)

Top-level is strictly topic-based: Get Started · Guides · Modules · Integrations · Administration · Reports · Reference · Release Notes. Roles & privileges still hide what the viewer can't access, but never appear in labels. Closest paradigm to ClickHouse, Stripe, Vercel.

Pros
  • Cleanest IA — single mental model regardless of role.
  • Strong SEO and external-search alignment (every doc has a topic home).
  • Adds a horizontal nav strip, freeing the sidebar for depth.
Cons
  • Requires a top nav strip + sidebar — small frontend lift.
  • Users used to "Admin Guide" must relearn that admin docs now live under Modules / Administration / Integrations.
  • Learners may feel the menu is "too much" until role filtering kicks in.

Option C · Highest polish

Persona-Led Landing + Topic Deep Dive

A custom landing page with three persona doors (Learner / Manager / Admin). Each door opens the same topic-based tree filtered to that audience. The sidebar deep-pages look like Option B — the difference is the front door.

Pros
  • Strongest "this site knows me" feeling on arrival.
  • Lets us highlight role-specific "What's new" and quick links.
  • Inside each persona, IA is still consistent (topic-first).
Cons
  • Needs a custom landing React component (swizzled / route).
  • Two clicks to start browsing — extra friction for power users.
  • If a user wears multiple hats, they may bounce between doors.
